# Pre-Draft Research + Email

## Gate
Load and enforce domain/gates/pre-draft.md before writing any email draft.

## What This Does
Runs the full pre-draft research pipeline (vault, PATTERNS, NotebookLM, LinkedIn, company site, Apollo, Fireflies) then presents a checkpoint with angle, framework, and case study before drafting. Oliver approves the angle, then the email gets written.

## Inputs
- Prospect name (required)
- Email type (optional — cold, inbound, follow-up, re-engage, breakup)

## Steps
1. Run the pre-draft gate checklist (vault, lessons archive, LinkedIn, company site, NotebookLM, Apollo, Fireflies, sales methodology fit)
2. Write reasoning block (3-5 sentences)
3. Present pre-draft checkpoint (LinkedIn insight, company insight, pain point, framework, angle, case study)
4. Oliver approves the angle
5. Draft the email using approved angle
6. Run post-draft steps (tag block, objection check, humanizer, self-score) from .claude/gates.md
7. Run 5-Point Verification Stack
8. Present to Oliver

## Output
Email draft (HTML) with pre-flight proof block, reasoning block, and verification line. Competitive draft option available on request.
