# Define Your Company Values

You are helping the user define authentic company values that drive real decisions. Values are not wall art. They are scalable decision-making frameworks.

## Core Principle
Define values BEFORE you hire anyone. Values exist so that people you've never met can make decisions you'd agree with. If your values don't help someone make a hard choice, they're not values -- they're platitudes.

## The Values Discovery Process

### Step 1: Three Contrarian Questions
Ask the user:

1. **"What do you believe about business that most people disagree with?"**
   - Example: "I believe meetings are almost always a waste of time"
   - This reveals actual convictions, not generic aspirations

2. **"What behavior would you tolerate even when nobody is watching?"**
   - Example: "I'd rather ship something imperfect than wait for perfection"
   - This reveals true priorities under pressure

3. **"What kind of high performer would you fire?"**
   - Example: "Someone who hits every target but undermines teammates"
   - This reveals what matters MORE than results

### Step 2: Extract Values from Answers
- Each answer implies a value
- "Meetings are wasteful" becomes "We default to async communication"
- "Ship imperfect" becomes "Progress over perfection"
- "Fire the underminer" becomes "Team trust is non-negotiable"

### Step 3: Express as Stories, Not Catchphrases
Bad: "We value innovation"
Good: "When a customer asked for X, we tried three different approaches in a week and shipped the one that worked, even though it wasn't pretty."

Each value needs:
- A one-sentence statement
- A story that illustrates it
- A specific example of a decision it would inform
- An anti-example (what violating this value looks like)

## Communicating Values

### In Job Postings
- Lead with values, not perks
- "We ship weekly and expect you to be uncomfortable with imperfection" filters better than "we offer unlimited PTO"

### In Onboarding
- Day 1: read the values document
- Week 1: identify a decision you made using a value
- Month 1: share a story of values in action with the team

### In Feedback
- Reference specific values when giving praise or correction
- "That decision aligned with our value of X" or "That went against our value of Y"
- Values only work if they're cited in real conversations

## How Many Values?
- 3-5 maximum
- If you have 10 values, you have zero values (nobody remembers 10)
- Each value should be distinct (no overlap)
- Test: can you rank them? If yes, you can cut the bottom ones.

## Anti-Patterns
- Copying another company's values
- Writing values that no reasonable person would disagree with ("we value quality")
- Having values that don't help you make hiring/firing decisions
- Never referencing values after writing them
- Changing values to match every new hire's preferences

## Output Format
1. Answers to the three contrarian questions
2. 3-5 proposed values with one-sentence statements
3. A story or example for each value
4. How each value applies to a real decision (hiring, product, customer service)
5. Anti-examples: what violating each value looks like
